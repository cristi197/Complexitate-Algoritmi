Pasul 1 · Fundamentul

Ce este eficiența unui algoritm?

Definiție simplă: Eficiența unui algoritm ne spune cât de rapid rezolvă o problemă și câtă memorie consumă, în funcție de dimensiunea datelor de intrare.

🌍 Analogie din viața reală

Imaginează-ți că trebuie să găsești un cuvânt într-o carte:

📖 Fără index

Citești pagina cu pagina până găsești. Dacă cartea are 1000 de pagini → 1000 de verificări. Ineficient.

🔍 Cu index (alfabetic)

Mergi direct la litera respectivă. 26 de litere → max ~10 pași. Eficient!

Exact aceeași logică se aplică algoritmilor: unii „citesc toată cartea", alții „folosesc indexul".

📐 De ce contează asta?

  • Un algoritm lent pe 10 elemente poate dura ani pe un milion de elemente.
  • Un program care consumă prea multă memorie se poate bloca pe calculatoare obișnuite.
  • La bacalaureat și la facultate ești evaluat pe corectitudinea analizei de complexitate.
Concluzie: Eficiența are două dimensiuni pe care trebuie să le justifici mereu:
→ Complexitatea timp (câte operații face algoritmul)
→ Complexitatea spațiu (câtă memorie suplimentară folosește)

🗂️ Termeni importanți

n

Dimensiunea datelor de intrare (ex: numărul de elemente dintr-un vector)

Operație elementară

O comparație, o atribuire, o adunare — ceva ce procesorul face „dintr-o lovitură"

Caz favorabil

Datele de intrare sunt aranjate cel mai bine posibil pentru algoritm

Caz defavorabil

Datele de intrare sunt cel mai rău posibil aranjate (worst case)

Caz mediu

Situația tipică, probabilă în practică

Pasul 2 · Măsurare

Cum măsurăm eficiența?

❌ Ce NU facem

Nu măsurăm timpul în secunde — asta depinde de calculator, limbaj, compilator și nu este o măsură matematică riguroasă.

Problemă: Același algoritm rulat pe un calculator din 2010 durează 10 secunde, iar pe unul din 2024 durează 0.3 secunde. Deci timpul real nu ne spune nimic despre algoritm în sine.

✅ Ce facem în schimb

Numărăm numărul de operații elementare în funcție de n (dimensiunea intrării).

  • Identificăm operația de bază

    De obicei comparația sau atribuirea care se repetă cel mai mult în algoritm.

  • Numărăm de câte ori se execută

    Exprimăm rezultatul ca funcție de n: f(n) = 3n² + 2n + 5

  • Simplificăm cu notația asimptotică

    Păstrăm doar termenul dominant, ignorăm constantele: O(n²)

💡 De ce simplificăm?

n = 1.000

3n² + 2n + 5 = 3.002.005
vs n² = 1.000.000
Diferența: ~200%. Mare!

n = 1.000.000

3n² domină absolut. Termenul 2n devine nesemnificativ (<0.0001%).

Pentru n mare, termenul dominant contează. Restul se ignoră în analiza asimptotică.

📊 Exemplu concret — căutarea unui element

// Câte comparații face acest cod?
for (i = 0; i < n; i++) {          // se execută n ori
    if (v[i] == target)  // 1 comparație × n = n comparații
        return i;
}
return -1;
// Total: n comparații → O(n)

🔴 Big-O — cel mai important la BAC și facultate

O(g(n)) înseamnă că algoritmul face cel mult c·g(n) operații pentru n suficient de mare (c este o constantă pozitivă).

La bacalaureat, aproape mereu vei folosi Big-O pentru cazul defavorabil.

📋 Reguli de calcul Big-O

Expresie originalăSimplificare Big-OMotivul
5n + 3O(n)Ignorăm constanta 5 și termenul liber 3
3n² + 7n + 100O(n²)Păstrăm doar termenul dominant n²
log₂n + nO(n)n crește mai repede decât log n
2ⁿ + n¹⁰⁰O(2ⁿ)Exponențialul domină orice polinomial
n! + 2ⁿO(n!)Factorialul domină exponențialul
42O(1)Constantă — nu depinde de n

⚡ Regula de aur: ordinea dominanței

De la cea mai bună la cea mai proastă complexitate:
O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

🔢 Exemplu practic cu reguli

// Câte operații?
for (i = 0; i < n; i++)         // n iterații
    for (j = 0; j < n; j++)     // n iterații × n = n²
        sum += v[i] * v[j];     // 1 operație

// Total: n × n = n² operații → O(n²)

// Secvential (nu imbricat):
for (i = 0; i < n; i++) x++;     // n operații
for (j = 0; j < n; j++) y++;     // n operații
// Total: n + n = 2n → O(n)  (nu O(2n)!)

Pasul 5 · Prima dimensiune

Complexitatea timp — Cum o calculezi

Definiție: Complexitatea timp T(n) exprimă numărul de operații elementare pe care le execută algoritmul în funcție de dimensiunea datelor de intrare n.

🔧 Reguli de calcul pentru structuri de bază

Instrucțiune simplă

O atribuire, comparație, operație aritmetică = O(1)

Secvență de instrucțiuni

Se adună complexitățile: O(1) + O(1) = O(1)

Structură if-else

Luăm maximul dintre cele două ramuri (caz defavorabil)

Buclă for/while

Se înmulțește numărul iterații × complexitatea corpului

🔄 Analiza buclelor — tipuri comune

// TIP 1: Buclă liniară → O(n)
for (i = 1; i <= n; i++)
    x = x + 1;               // se execută exact n ori → O(n)

// TIP 2: Buclă pătratică → O(n²)
for (i = 1; i <= n; i++)
    for (j = 1; j <= n; j++)
        x = x + 1;           // n × n = n² ori → O(n²)

// TIP 3: Buclă logaritmică → O(log n)
i = 1;
while (i < n)
    i = i * 2;               // i: 1,2,4,8,...,n → log₂n pași → O(log n)

// TIP 4: Buclă care înjumătățește → O(log n)
while (n > 1)
    n = n / 2;               // 1000→500→250→...→1 (~10 pași) → O(log n)

// TIP 5: Triunghiular → O(n²)
for (i = 1; i <= n; i++)
    for (j = i+1; j <= n; j++) // 0+1+2+...+(n-1) = n(n-1)/2 → O(n²)
        x++;

📌 Exemplu complet: căutare binară

int cautareBinara(int v[], int n, int target) {
    int st = 0, dr = n - 1;
    while (st <= dr) {               // se execută de câte ori?
        int mij = (st + dr) / 2;    // intervalul se înjumătățește la fiecare pas
        if (v[mij] == target)
            return mij;
        else if (v[mij] < target)
            st = mij + 1;
        else
            dr = mij - 1;
    }                                // interval: n → n/2 → n/4 → ... → 1
    return -1;                      // Număr de pași: log₂n → O(log n)
}
Justificare complexitate timp (model BAC):
La fiecare iterație a buclei while, intervalul de căutare se reduce la jumătate. Dacă initial intervalul are n elemente, după k pași rămân n/2ᵏ elemente. Algoritmul se oprește când n/2ᵏ ≤ 1, adică k ≥ log₂n. Deci numărul de iterații este ⌊log₂n⌋ + 1 → O(log n).

Pasul 6 · A doua dimensiune

Complexitatea spațiu — La fel de importantă!

Definiție: Complexitatea spațiu S(n) exprimă cantitatea de memorie suplimentară pe care o folosește algoritmul (pe lângă datele de intrare), în funcție de n.
Important! La BAC și facultate, complexitatea spațiu se referă la memoria auxiliară — adică extra față de vectorul dat. Vectorul de intrare nu se pune în calcul (este deja acolo).

📦 Tipuri de memorie suplimentară

Variabile simple

int i, j, x;
Fiecare variabilă = O(1) spațiu. Oricâte variabile simple = tot O(1).

Array auxiliar

int temp[n];
Un vector de n elemente = O(n) spațiu suplimentar.

Matrice auxiliară

int mat[n][n];
O matrice n×n = O(n²) spațiu suplimentar.

Stivă recursivă

La fiecare apel recursiv se alocă spațiu pe stivă. Adâncimea recursivității = spațiu suplimentar.

🔍 Exemple concrete

// EXEMPLUL 1: O(1) spațiu suplimentar
void suma(int v[], int n) {
    int s = 0;          // doar o variabilă → O(1)
    for (int i = 0; i < n; i++)
        s += v[i];
    return s;
}  // Spațiu suplimentar: s, i → O(1)

// EXEMPLUL 2: O(n) spațiu suplimentar
void copiaza(int v[], int n) {
    int copie[n];        // vector auxiliar de n elemente → O(n)
    for (int i = 0; i < n; i++)
        copie[i] = v[i];
}  // Spațiu suplimentar: copie[n] → O(n)

// EXEMPLUL 3: Recursivitate → spațiu = adâncimea stivei
int factorial(int n) {
    if (n == 0) return 1;
    return n * factorial(n - 1);
}
// Apeluri: factorial(n) → factorial(n-1) → ... → factorial(0)
// Adâncime: n apeluri pe stivă → O(n) spațiu suplimentar

⚖️ Trade-off timp-spațiu (foarte important!)

De multe ori există un compromis între complexitatea timp și cea spațiu:

Varianta rapidă (mai mult spațiu)

Memorăm rezultate intermediare (memoizare, tabelare) → mai puțin timp, mai mult spațiu

Varianta economică (mai lent)

Recalculăm la nevoie → mai puțin spațiu, mai mult timp

Model de justificare spațiu (BAC):
"Algoritmul folosește variabilele i, j, aux (spațiu constant O(1)) și nu alocă structuri suplimentare proporționale cu n. Prin urmare, complexitatea spațiu este O(1)."

Pasul 8 · Metodologie

Cum analizezi complexitatea — Metodă pas cu pas

Urmând acești pași vei putea analiza corect orice algoritm dat la bacalaureat sau la examenele de facultate.
  • Identifică variabila n

    Ce reprezintă dimensiunea datelor de intrare? (nr. elemente din vector, valoarea unui număr, dimensiunea matricei etc.)

  • Identifică operația elementară dominantă

    De obicei este comparația sau atribuirea care se execută cel mai des. Asta va da termenul principal.

  • Numără buclele și nivelurile de imbricare

    O buclă = O(n). Două bucle imbricate = O(n²). Trei bucle imbricate = O(n³). Buclă care dublează/înjumătățește = O(log n).

  • Analizează cazurile (favorabil / defavorabil / mediu)

    Gândește-te: "Când se termină cel mai repede? Când cel mai târziu?"

  • Simplifică cu Big-O

    Ia termenul dominant, ignoră constantele și termenii de ordin inferior.

  • Analizează spațiul suplimentar

    Numără variabilele auxiliare, vectorii și matricele alocate suplimentar. Adaugă adâncimea stivei dacă e recursiv.

  • Redactează justificarea

    Scrie complet: "Algoritmul efectuează aproximativ f(n) operații deoarece... Prin urmare complexitatea timp este O(...)."

🖊️ Modele de justificare scrisă (pentru BAC)

Model 1 — Buclă simplă O(n):
"Bucla for se execută de n ori, efectuând la fiecare pas un număr constant de operații elementare. Deci numărul total de operații este proporțional cu n. Complexitatea timp este O(n). Algoritmul folosește doar variabilele i și s, deci complexitatea spațiu este O(1)."
Model 2 — Bucle imbricate O(n²):
"Bucla exterioară se execută de n ori. La fiecare iterație a buclei exterioare, bucla interioară se execută de n ori. Total: n × n = n² operații elementare. Complexitatea timp este O(n²). Nu se alocă memorie suplimentară în afara variabilelor temporare, deci complexitatea spațiu este O(1)."
Model 3 — Recursivitate O(log n):
"La fiecare apel recursiv, dimensiunea problemei se înjumătățește. Dacă pornim de la n, după k apeluri avem n/2ᵏ = 1, deci k = log₂n. Complexitatea timp este O(log n). Adâncimea stivei de recursivitate este de log₂n apeluri, deci complexitatea spațiu este O(log n)."

Pasul 9 · Algoritmi în detaliu

Algoritmi de sortare — Analiză detaliată

🫧 Bubble Sort — Sortare prin interschimbare

void bubbleSort(int v[], int n) {
    for (int i = 0; i < n-1; i++) {        // n-1 treceri
        for (int j = 0; j < n-i-1; j++) {  // treceri înjumătățite
            if (v[j] > v[j+1]) {
                int aux = v[j];              // interschimbare
                v[j] = v[j+1];
                v[j+1] = aux;
            }
        }
    }
}
// Comparații: (n-1)+(n-2)+...+1 = n(n-1)/2 → O(n²)
// Spațiu: aux (O(1))
Justificare timp: Bucla exterioară face n-1 iterații. La iterația i, bucla interioară face n-i-1 comparații. Total: Σ(n-i-1) pentru i=0..n-2 = (n-1)+(n-2)+...+1 = n(n-1)/2 ≈ n²/2 → O(n²).

🎯 Selection Sort — Sortare prin selecție

void selectionSort(int v[], int n) {
    for (int i = 0; i < n-1; i++) {
        int minIdx = i;
        for (int j = i+1; j < n; j++)  // caută minimul
            if (v[j] < v[minIdx])
                minIdx = j;
        // interschimbă minimul găsit cu v[i]
        int aux = v[minIdx]; v[minIdx] = v[i]; v[i] = aux;
    }
}
// Comparații: exact n(n-1)/2 indiferent de date → O(n²) în orice caz

📥 Insertion Sort — Sortare prin inserție

void insertionSort(int v[], int n) {
    for (int i = 1; i < n; i++) {
        int key = v[i];
        int j = i - 1;
        while (j >= 0 && v[j] > key) {  // deplasează elementele mai mari
            v[j+1] = v[j];
            j--;
        }
        v[j+1] = key;
    }
}
// Favorabil (sortat): O(n) — while nu se execută niciodată
// Defavorabil (sortat invers): O(n²) — while face i pași la fiecare i

⚡ Merge Sort — Sortare prin interclasare

void mergeSort(int v[], int st, int dr) {
    if (st < dr) {
        int mij = (st + dr) / 2;
        mergeSort(v, st, mij);      // ↙ stânga
        mergeSort(v, mij+1, dr);   // ↘ dreapta
        interclasare(v, st, mij, dr); // combină în O(n)
    }
}
// Recurență: T(n) = 2·T(n/2) + O(n)
// Prin teorema master: O(n log n) în orice caz!
// Spațiu: O(n) pentru vectorul auxiliar de interclasare
De ce O(n log n)? La fiecare nivel de recursivitate se fac în total n comparații (interclasare). Numărul de niveluri este log₂n (jumătățim de fiecare dată). Deci total: n × log₂n operații → O(n log n).

Pasul 10 · Exersează!

Probleme tip BAC cu rezolvări complete

Acestea sunt tipuri de cerințe des întâlnite la bacalaureatul de informatică în România. Studiaza modelul de răspuns!

🟦 Problema 1 — Numărare (Facile)

Se dă următoarea secvență de cod. Determinați complexitatea timp și spațiu justificând răspunsul.

int contor = 0;
for (int i = 0; i < n; i++)
    for (int j = i; j < n; j++)
        contor++;
Rezolvare:
Complexitate timp: Bucla exterioară face n iterații (i = 0,1,...,n-1). Pentru fiecare valoare i, bucla interioară face n-i iterații. Numărul total de execuții al instrucțiunii contor++ este:
Σᵢ₌₀ⁿ⁻¹ (n-i) = n + (n-1) + (n-2) + ... + 1 = n(n+1)/2
Prin urmare, complexitatea timp este O(n²).

Complexitate spațiu: Se alocă doar variabilele contor, i, j — un număr constant de variabile, independent de n. Complexitatea spațiu este O(1).

🟧 Problema 2 — Recursivitate (Medie)

Analizați complexitatea funcției recursive de mai jos:

int putere(int a, int n) {
    if (n == 0) return 1;
    if (n % 2 == 0)
        return putere(a * a, n / 2);
    else
        return a * putere(a, n - 1);
}
Rezolvare:
Complexitate timp: Observăm că la fiecare apel, exponentul n se înjumătățește (dacă e par) sau scade cu 1 (dacă e impar). În cel mai rău caz, scăderea cu 1 face exponentul par, deci după maximum 2 apeluri, exponentul se înjumătățește. Numărul de apeluri este deci proporțional cu log₂n. Complexitatea timp este O(log n).

Complexitate spațiu: Fiecare apel recursiv adaugă un cadru pe stivă. Adâncimea maximă a recursivității este O(log n). Deci complexitatea spațiu este O(log n).

🟥 Problema 3 — Comparație algoritmi (Dificilă / BAC nivel înalt)

Se dă un vector cu n = 100.000 elemente întregi, nevoi să verifici dacă un element k se află în vector. Ai la dispoziție două variante:

  • Varianta A: Căutare secvențială pe vectorul nesortat
  • Varianta B: Sortezi vectorul cu Merge Sort, apoi aplici căutare binară

Care variantă este mai eficientă pentru o singură interogare? Dar pentru 1.000 de interogări? Justificați.

Rezolvare:
Varianta A — O interogare: Căutarea secvențială face O(n) comparații. Pentru n=100.000 → ~100.000 operații.

Varianta B — O interogare: Merge Sort → O(n log n) = 100.000 × 17 ≈ 1.700.000 operații, plus căutare binară O(log n) ≈ 17 operații. Total: ~1.700.017 operații.
→ Pentru o singură interogare, Varianta A este mai eficientă.

Varianta B — 1.000 de interogări: Sortăm o singură dată: 1.700.000 operații. Fiecare din cele 1.000 de căutări binare: 17 operații. Total: 1.700.000 + 1.000×17 = 1.717.000 operații.
Varianta A — 1.000 de interogări: 1.000 × 100.000 = 100.000.000 operații.
→ Pentru 1.000 de interogări, Varianta B este de ~58 de ori mai eficientă!

🟩 Problema 4 — Identificare complexitate din cod (BAC clasic)

Care este complexitatea timp a algoritmului următor? Justificați.

int s = 0;
for (int i = 1; i <= n; i *= 2)       // i: 1, 2, 4, 8, ..., 2^k
    for (int j = 1; j <= n; j++)
        s += i + j;
Rezolvare:
Bucla exterioară: i ia valorile 1, 2, 4, ..., 2^⌊log₂n⌋ → se execută de ⌊log₂n⌋+1 ≈ log₂n ori.
Bucla interioară: pentru fiecare valoare a lui i, j parcurge 1..n → se execută de n ori.
Total operații: log₂n × n = n·log n.
Complexitatea timp este O(n log n).
Complexitatea spațiu: variabilele s, i, j → O(1).

Sfaturi esențiale pentru BAC și facultate

✅ Ce trebuie să știi obligatoriu

Complexitățile de bază

O(1), O(log n), O(n), O(n log n), O(n²), O(2ⁿ) — știi să le recunoști și să le justifici.

Cele două dimensiuni

Întotdeauna justifică AMBELE: complexitate timp și complexitate spațiu.

Cele 3 cazuri

Menționează cazul favorabil, mediu și defavorabil (cel puțin defavorabilul la BAC).

Regulile de simplificare

Ignoră constantele. Ignoră termenii de ordin inferior. Păstrează termenul dominant.

❌ Greșeli frecvente

  • ❌ Să spui că un algoritm e "rapid" sau "lent" fără justificare matematică
  • ❌ Să uiți de complexitatea spațiu
  • ❌ Să confunzi O(log n) cu O(n) la bucle care înjumătățesc
  • ❌ Să consideri că O(2n) este altceva decât O(n)
  • ❌ Să nu specifici față de ce variabilă (n) este complexitatea
  • ❌ La recursivitate, să uiți spațiul stivei de recursivitate
